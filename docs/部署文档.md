## 方式一：集成到 Vibe Kanban

这是最常见的使用场景 — 为 Vibe Kanban 添加密码保护。

### 步骤 1：修改依赖

在 `vibe-kanban/crates/server/Cargo.toml` 中添加 auth-wall 依赖：

```toml
[dependencies]
auth-wall = { git = "https://github.com/yangfei957/vibe-kanban-plus.git", optional = true }

[features]
default = []
auth-wall = ["dep:auth-wall"]
```

> **注意：** 确保 `crates/auth-wall` 目录**不在** workspace members 中（应已移除）。

### 步骤 2：添加集成代码

在 `crates/server/src/routes/mod.rs` 中添加条件编译块：

```rust
// 在路由构建完成后、返回前添加：

// 当 auth-wall feature 启用时，包装整个应用
#[cfg(feature = "auth-wall")]
let app = {
    let auth_state = auth_wall_state();
    app.merge(auth_wall::auth_wall_routes(auth_state.clone()))
        .layer(axum::middleware::from_fn_with_state(
            auth_state,
            auth_wall::auth_wall_middleware,
        ))
};
```

以及配置构建函数：

```rust
/// 从环境变量构建 auth-wall 状态
#[cfg(feature = "auth-wall")]
fn auth_wall_state() -> auth_wall::AuthWallState {
    let password_path = std::env::var("AUTH_WALL_PASSWORD_FILE")
        .map(std::path::PathBuf::from)
        .unwrap_or_else(|_| {
            // 使用你的应用数据目录
            let data_dir = your_app::data_dir();
            data_dir.join("auth_wall_password.hash")
        });

    let max_attempts = std::env::var("AUTH_WALL_MAX_ATTEMPTS")
        .ok().and_then(|s| s.parse().ok()).unwrap_or(5);

    let lockout_secs = std::env::var("AUTH_WALL_LOCKOUT_SECS")
        .ok().and_then(|s| s.parse().ok()).unwrap_or(300);

    let session_secs = std::env::var("AUTH_WALL_SESSION_SECS")
        .ok().and_then(|s| s.parse().ok()).unwrap_or(86400);

    let secure_cookie = std::env::var("AUTH_WALL_SECURE_COOKIE")
        .map(|s| s == "1" || s.eq_ignore_ascii_case("true"))
        .unwrap_or(false);

    let config = auth_wall::AuthWallConfig {
        password_hash_path: password_path,
        max_failed_attempts: max_attempts,
        lockout_duration_secs: lockout_secs,
        session_expiry_secs: session_secs,
        secure_cookie,
    };

    auth_wall::AuthWallState::new(config)
}
```

### 步骤 3：设置密码

```bash
# 编译并运行密码设置工具
cargo run --bin set-password
```

交互过程：

```
╔══════════════════════════════════════════╗
║     Vibe Kanban — Auth Wall Setup        ║
╚══════════════════════════════════════════╝

Enter new password: ********
Confirm password: ********

✅ Password has been set successfully!
   Hash file: /home/user/.local/share/ai.bloop.vibe-kanban/auth_wall_password.hash

ℹ  Restart Vibe Kanban for the auth wall to take effect.
```

也可指定自定义路径：

```bash
AUTH_WALL_PASSWORD_FILE=/opt/vibe-kanban/password.hash cargo run --bin set-password
```

### 步骤 4：启动带认证的服务

```bash
cargo run --bin server --features auth-wall
```

### 步骤 5：验证部署

```bash
# 1. 访问应用 — 应被重定向到登录页
curl -v http://localhost:8080/
# 预期：302 Found, Location: /auth-wall/login

# 2. 检查登录页面
curl http://localhost:8080/auth-wall/login
# 预期：200 OK, HTML 内容

# 3. API 请求应被拒绝
curl http://localhost:8080/api/config
# 预期：401 Unauthorized

# 4. 查询认证状态
curl http://localhost:8080/auth-wall/api/status
# 预期：{"authenticated":false,"password_configured":true}

# 5. 登录
curl -X POST http://localhost:8080/auth-wall/api/login \
  -H "Content-Type: application/json" \
  -d '{"password":"your_password"}' \
  -c cookies.txt
# 预期：{"success":true,"message":"Login successful"}

# 6. 使用 Cookie 访问
curl -b cookies.txt http://localhost:8080/
# 预期：200 OK, 正常页面内容
```

## 方式二：集成到其他 Axum 项目

Auth Wall 完全独立于 Vibe Kanban，可用于任何 Axum 0.8+ 应用。

### 步骤 1：添加依赖

```toml
# Cargo.toml
[dependencies]
auth-wall = { git = "https://github.com/yangfei957/vibe-kanban-plus.git" }
axum = "0.8"
tokio = { version = "1", features = ["full"] }
```

### 步骤 2：集成到你的应用

```rust
use auth_wall::{AuthWallConfig, AuthWallState, auth_wall_routes, auth_wall_middleware};
use axum::{Router, routing::get, middleware};

#[tokio::main]
async fn main() {
    // 配置 auth-wall
    let config = AuthWallConfig {
        password_hash_path: "/data/my_app_password.hash".into(),
        max_failed_attempts: 5,
        lockout_duration_secs: 300,
        session_expiry_secs: 86400,
        secure_cookie: false,
    };
    let auth_state = AuthWallState::new(config);

    // 构建路由
    let app = Router::new()
        .route("/", get(|| async { "Hello, World!" }))
        // ... 你的其他路由
        .merge(auth_wall_routes(auth_state.clone()))           // 添加登录页面路由
        .layer(middleware::from_fn_with_state(
            auth_state,
            auth_wall_middleware,                                // 添加认证中间件
        ));

    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}
```

### 步骤 3：设置密码并运行

```bash
AUTH_WALL_PASSWORD_FILE=/data/my_app_password.hash cargo run --bin set-password
cargo run
```

## 方式三：Docker 容器部署

### Dockerfile 示例

```dockerfile
# ---- 构建阶段 ----
FROM rustlang/rust:nightly AS builder

WORKDIR /build

# 先复制并构建依赖（利用 Docker 缓存）
COPY Cargo.toml Cargo.lock ./
RUN mkdir src && echo "fn main() {}" > src/main.rs && \
    cargo build --release 2>/dev/null || true
RUN rm -rf src

# 复制源代码并构建
COPY . .
RUN cargo build --release --bin server --features auth-wall && \
    cargo build --release --bin set-password

# ---- 运行阶段 ----
FROM debian:bookworm-slim

RUN apt-get update && \
    apt-get install -y --no-install-recommends ca-certificates && \
    rm -rf /var/lib/apt/lists/*

# 创建非 root 用户
RUN useradd -m -s /bin/bash appuser

COPY --from=builder /build/target/release/server /usr/local/bin/
COPY --from=builder /build/target/release/set-password /usr/local/bin/

# 数据卷
RUN mkdir -p /data && chown appuser:appuser /data
VOLUME /data

ENV AUTH_WALL_PASSWORD_FILE=/data/auth_wall_password.hash

USER appuser
EXPOSE 8080
CMD ["server"]
```

### 构建和运行

```bash
# 构建镜像
docker build -t vibe-kanban:auth .

# 首次设置密码（交互模式）
docker run -it --rm \
  -v vibe-data:/data \
  vibe-kanban:auth \
  set-password

# 启动服务
docker run -d \
  --name vibe-kanban \
  -p 8080:8080 \
  -v vibe-data:/data \
  --restart unless-stopped \
  vibe-kanban:auth
```

### Docker Compose 示例

```yaml
# docker-compose.yml
version: "3.8"

services:
  vibe-kanban:
    image: vibe-kanban:auth
    ports:
      - "8080:8080"
    volumes:
      - vibe-data:/data
    environment:
      - AUTH_WALL_PASSWORD_FILE=/data/auth_wall_password.hash
      - AUTH_WALL_MAX_ATTEMPTS=5
      - AUTH_WALL_LOCKOUT_SECS=300
      - AUTH_WALL_SESSION_SECS=86400
      - AUTH_WALL_SECURE_COOKIE=false
    restart: unless-stopped

volumes:
  vibe-data:
```

```bash
# 首次设置密码
docker compose run --rm vibe-kanban set-password

# 启动
docker compose up -d

# 查看日志
docker compose logs -f vibe-kanban
```

## 方式四：反向代理 + HTTPS 生产部署

生产环境推荐使用反向代理提供 HTTPS，并启用 Secure Cookie。

### Caddy（推荐，自动 HTTPS）

```caddyfile
# Caddyfile
yourdomain.com {
    reverse_proxy localhost:8080
}
```

```bash
# 启动时启用 Secure Cookie
AUTH_WALL_SECURE_COOKIE=true cargo run --bin server --features auth-wall
```

Caddy 会自动申请和续期 Let's Encrypt 证书。

### Nginx

```nginx
server {
    listen 443 ssl http2;
    server_name yourdomain.com;

    ssl_certificate     /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$server_name$request_uri;
}
```

> **重要：** Auth Wall 使用 `X-Forwarded-For` 头获取客户端真实 IP 以进行暴力破解防护。
> 请确保反向代理正确设置此头。

### systemd 服务

```ini
# /etc/systemd/system/vibe-kanban.service
[Unit]
Description=Vibe Kanban with Auth Wall
After=network.target

[Service]
Type=simple
User=vibe-kanban
Group=vibe-kanban
WorkingDirectory=/opt/vibe-kanban
ExecStart=/opt/vibe-kanban/server
Restart=always
RestartSec=5

# 环境变量
Environment=AUTH_WALL_PASSWORD_FILE=/opt/vibe-kanban/data/password.hash
Environment=AUTH_WALL_SECURE_COOKIE=true
Environment=AUTH_WALL_MAX_ATTEMPTS=5
Environment=AUTH_WALL_LOCKOUT_SECS=300
Environment=AUTH_WALL_SESSION_SECS=86400

# 安全加固
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/opt/vibe-kanban/data

[Install]
WantedBy=multi-user.target
```

```bash
# 创建系统用户
sudo useradd -r -s /usr/sbin/nologin vibe-kanban

# 安装二进制文件
sudo mkdir -p /opt/vibe-kanban/data
sudo cp target/release/server /opt/vibe-kanban/
sudo cp target/release/set-password /opt/vibe-kanban/
sudo chown -R vibe-kanban:vibe-kanban /opt/vibe-kanban

# 设置密码
sudo -u vibe-kanban AUTH_WALL_PASSWORD_FILE=/opt/vibe-kanban/data/password.hash \
  /opt/vibe-kanban/set-password

# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable --now vibe-kanban

# 查看状态
sudo systemctl status vibe-kanban
sudo journalctl -u vibe-kanban -f
```

## 配置参考

### 环境变量一览

| 变量名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `AUTH_WALL_PASSWORD_FILE` | 文件路径 | `<data_dir>/auth_wall_password.hash` | 密码哈希文件存储路径 |
| `AUTH_WALL_MAX_ATTEMPTS` | 正整数 | `5` | 单个 IP 锁定前最大失败尝试次数 |
| `AUTH_WALL_LOCKOUT_SECS` | 正整数 | `300`（5 分钟） | IP 锁定持续时间（秒） |
| `AUTH_WALL_SESSION_SECS` | 正整数 | `86400`（24 小时） | 登录会话有效期（秒） |
| `AUTH_WALL_SECURE_COOKIE` | `true`/`false`/`1`/`0` | `false` | 是否在 Cookie 上设置 Secure 标记 |

### 默认数据目录

`AUTH_WALL_PASSWORD_FILE` 未设置时，`set-password` 工具使用以下默认路径：

| 操作系统 | 默认路径 |
|----------|----------|
| **Linux** | `~/.local/share/ai.bloop.vibe-kanban/auth_wall_password.hash` |
| **macOS** | `~/Library/Application Support/ai.bloop.vibe-kanban/auth_wall_password.hash` |
| **Windows** | `C:\Users\<用户>\AppData\Roaming\bloop\vibe-kanban\data\auth_wall_password.hash` |

### 推荐配置

#### 本地开发

```bash
# 无需额外配置，直接运行
cargo run --bin set-password
cargo run --bin server --features auth-wall
```

#### 内网部署

```bash
export AUTH_WALL_PASSWORD_FILE=/opt/vibe-kanban/password.hash
export AUTH_WALL_SESSION_SECS=604800   # 7 天会话
```

#### 公网生产环境

```bash
export AUTH_WALL_PASSWORD_FILE=/opt/vibe-kanban/password.hash
export AUTH_WALL_SECURE_COOKIE=true     # 必须启用（HTTPS）
export AUTH_WALL_MAX_ATTEMPTS=3         # 更严格的限制
export AUTH_WALL_LOCKOUT_SECS=600       # 10 分钟锁定
export AUTH_WALL_SESSION_SECS=28800     # 8 小时会话
```

## 运维操作

### 修改密码

```bash
# 1. 运行 set-password 工具
AUTH_WALL_PASSWORD_FILE=/path/to/password.hash cargo run --bin set-password
# 或直接运行编译好的二进制文件
/opt/vibe-kanban/set-password

# 2. 重启服务以生效（已登录的会话不受影响，直到过期）
sudo systemctl restart vibe-kanban
```

### 禁用认证

有两种方式：

**方式 A — 删除密码文件（推荐）：**
```bash
rm /path/to/auth_wall_password.hash
# 无需重启，下一个请求即生效（因为中间件每次检查文件是否存在）
```

**方式 B — 不启用 feature 编译：**
```bash
cargo run --bin server   # 不加 --features auth-wall
```

### 查看日志

Auth Wall 通过 `tracing` 输出结构化日志：

```bash
# 查看认证相关日志
journalctl -u vibe-kanban | grep "Auth-wall"

# 典型日志输出：
# INFO  Auth-wall enabled. Password hash: /opt/.../password.hash  Max attempts: 5  Lockout: 300s  Session: 86400s
# INFO  Auth-wall: successful login from 192.168.1.100
# WARN  Auth-wall: failed login from 203.0.113.50 (3/5)
# WARN  Auth-wall: login attempt from locked-out IP: 203.0.113.50
```

### 解锁被锁定的 IP

当前版本不提供手动解锁命令。锁定会在配置的时间（默认 5 分钟）后自动解除。

如需立即解锁，重启服务即可（内存中的锁定状态会被清除）：

```bash
sudo systemctl restart vibe-kanban
```

## 故障排查

### 问题 1：访问页面不跳转到登录

**可能原因：**
- 编译时未启用 `auth-wall` feature → 确认使用 `--features auth-wall`
- 密码文件不存在或为空 → 运行 `set-password` 设置密码
- 密码文件路径不正确 → 检查 `AUTH_WALL_PASSWORD_FILE` 环境变量

**排查命令：**
```bash
# 检查密码文件
cat $AUTH_WALL_PASSWORD_FILE
# 应该输出 $argon2id$... 格式的哈希

# 检查是否带 feature 编译
cargo build --features auth-wall 2>&1 | head -5
```

### 问题 2：登录后立即被踢出

**可能原因：**
- Cookie 未正确设置 → 检查浏览器开发者工具 Application > Cookies
- HTTPS 环境但未启用 Secure Cookie → 设置 `AUTH_WALL_SECURE_COOKIE=true`
- 反之，非 HTTPS 但启用了 Secure Cookie → 设置 `AUTH_WALL_SECURE_COOKIE=false`

### 问题 3：被锁定无法登录

**原因：** 连续失败尝试超过阈值。

**解决方案：**
1. 等待锁定自动解除（查看 429 响应中的等待秒数）
2. 重启服务清除内存中的锁定状态
3. 增大 `AUTH_WALL_MAX_ATTEMPTS` 值

### 问题 4：忘记密码

**解决方案：**
```bash
# 重新设置密码
AUTH_WALL_PASSWORD_FILE=/path/to/password.hash set-password
# 重启服务
sudo systemctl restart vibe-kanban
```

### 问题 5：反向代理后所有用户共享同一个 IP

**原因：** 反向代理未正确传递 `X-Forwarded-For` 头，导致所有请求看起来来自同一 IP。

**解决方案：** 在反向代理配置中添加：

```nginx
# Nginx
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

```caddyfile
# Caddy — 默认已设置，无需额外配置
```

## 安全最佳实践

1. **始终使用 HTTPS** — 在生产环境中配合 Caddy 或 Nginx + Let's Encrypt
2. **启用 Secure Cookie** — HTTPS 环境下设置 `AUTH_WALL_SECURE_COOKIE=true`
3. **保护密码文件** — 设置适当的文件权限 `chmod 600 password.hash`
4. **使用强密码** — 至少 12 个字符，混合大小写字母、数字和特殊字符
5. **适当设置会话有效期** — 公网环境建议 8 小时，内网可适当延长
6. **定期检查日志** — 关注失败登录尝试和锁定事件
7. **保持更新** — 关注仓库更新以获取安全补丁
